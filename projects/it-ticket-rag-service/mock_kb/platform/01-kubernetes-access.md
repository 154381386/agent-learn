## 适用范围
适用于通过公司统一身份系统访问以下集群：
- `staging-shanghai-1`
- `prod-shanghai-1`
- `prod-beijing-1`

不适用于以下场景：
- 管理平面故障处理
- etcd 级别操作
- 节点 OS 层面的运维操作
- 云厂商底层资源修复

## 背景说明
公司内部所有 Kubernetes 集群均采用“短期凭证 + RBAC + 审计留痕”的访问模式。
研发默认可访问预发环境，生产环境采用最小权限原则，默认只开放只读能力，写操作必须依赖变更单或事故单。

|---|---|---|---|---|

## 访问前置条件
- 已接入办公网络或 VPN
- 已完成企业 SSO 登录
- 已开通堡垒机与平台门户账号
- 已通过集群访问审批
- 本机已安装 `kubectl >= 1.29`
- 本机时间同步正常，避免 token 校验失败

## 权限模型
### 研发默认权限
- `staging`：namespace 级别读写
- `prod`：默认只读

### 值班/SRE 默认权限
- `prod`：资源读取、日志查看、受控写操作
- 高风险动作需审批

### 不允许的默认权限
- cluster-admin 长期下发到个人账号
- 跨环境共用 kubeconfig 长期缓存
- 未审计的跳板直连

## kubeconfig 获取流程
1. 登录 `https://devops.internal.example.com`
2. 打开“集群访问”页面
3. 选择目标环境和角色
4. 下载短期 `kubeconfig`，有效期默认 12 小时
5. 保存到 `~/.kube/config` 或合并到现有配置
6. 使用 `kubectl config get-contexts` 校验

## 推荐配置方式
### 单环境临时覆盖
```bash
export KUBECONFIG=~/Downloads/prod-shanghai-1.kubeconfig
kubectl config current-context
```

### 多环境合并
```bash
KUBECONFIG=~/.kube/config:~/Downloads/prod-shanghai-1.kubeconfig kubectl config view --flatten > /tmp/merged-config
mv /tmp/merged-config ~/.kube/config
```

## 生产环境操作约束
- 禁止把生产环境 context 设为默认 context 后长期不切回
- 禁止直接 `kubectl delete pod` 作为首选修复手段
- 禁止绕过发布平台直接修改生产 Deployment 镜像 tag
- 禁止使用个人机器保存长期有效 kubeconfig
- 禁止未授权人员对 `kube-system` 资源做写操作

## 推荐排查习惯
1. 先确认当前 context，避免误连生产
2. 先读再写，先看 `get/describe/logs/top`
3. 查故障时同时记录：时间、集群、namespace、服务名
4. 对生产问题，优先结合监控与最近发布记录判断
5. 所有写操作要能关联到事故单或变更单

## 常见报错与处理
### 1. `You must be logged in to the server`
可能原因：
- kubeconfig 已过期
- token 被撤销
- 当前网络不在办公网/VPN

处理方式：
- 重新下载 kubeconfig
- 检查 VPN
- 重新登录开发门户

### 2. `forbidden`
可能原因：
- 当前角色无 namespace 权限
- 资源类型不在 RBAC 允许范围内
- 目标环境写权限未审批

处理方式：
- 确认是否访问了正确 cluster/context
- 提交权限申请
- 联系平台值班人确认角色绑定

### 3. `Unable to connect to the server`
可能原因：
- VPN 中断
- apiserver 网络异常
- 本地 DNS 异常

建议：
- 先用浏览器/内部网络工具验证控制面连通性
- 若多个同事同时异常，升级到平台值班

## FAQ
### Q1：研发能不能直接访问生产？
可以，但默认只读。写操作需要审批与审计。

### Q2：为什么 kubeconfig 只有 12 小时？
为了降低凭证泄露风险，生产环境不允许长期 token。

### Q3：我只是查日志，为什么也要走堡垒机/VPN？
因为集群 API 和观测平台都属于内网受控资源，统一要求通过公司网络访问。

## 升级条件
满足任一情况，建议立即拉值班 SRE：
- 需要执行生产写操作
- 需要跨 namespace / 跨集群联合排查
- 怀疑 apiserver / RBAC / admission controller 异常
- 多个服务同时访问失败，疑似平台级问题

## 安全与审计细则
- 所有生产 kubeconfig 下载行为都会记录账号、IP、时间和集群名
- 若 24 小时内多次下载不同生产集群凭证，会触发安全审计提醒
- 堡垒机退出后不等于 kubeconfig 失效，仍需注意本地文件清理
